# 攻撃用OSの比較: Kali Linux vs. Parrot OS

はい、Kali LinuxとParrot OSは、どちらもペネトレーションテストやセキュリティ監査などの用途に特化したLinuxディストリビューションです。しかし、いくつかの異なる特徴やメリット・デメリットがあります。

---

## Kali Linux

### メリット
- **広範なツール数**  
  セキュリティやペンテスト用のツールが非常に豊富で、デフォルトで600以上のツールがプリインストールされています。
- **公式ドキュメント・コミュニティの充実**  
  利用者が多く、情報がインターネット上にも豊富に存在します。困った時に解決しやすいです。
- **公式サポート**  
  Offensive Securityによる開発・サポートが継続的に行われているため、信頼性があります。
- **カスタマイズのしやすさ**  
  必要に応じてインストール時にツールセットなどをカスタマイズしやすい設計です。

### デメリット
- **プライバシー保護機能は限定的**  
  匿名性やプライバシー重視の機能は標準搭載していません。
- **システム要件がやや高い**  
  軽量ではありますが、Parrot OSのLite版などに比べてやや重い場合があります。
- **ペンテスト以外の用途に不向き**  
  日常利用向きではなく、セキュリティテスト特化のため普段使いには不便な面もあります。

---

## Parrot OS

### メリット
- **プライバシー・匿名性重視**  
  匿名ブラウジングやTor、sandboxなど、プライバシー保護や匿名性に配慮した機能が標準搭載されています。
- **多目的利用可能**  
  Securityエディション（ペンテスト用）のほか、Homeエディション（普段使い用）なども提供されており、日常利用にも適しています。
- **システム要件が低め**  
  軽量な設計で、古いマシンやリソースの少ない環境でも動作しやすいです。
- **独自のツールセット**  
  開発・プログラミング向けツールなども豊富に用意されています。

### デメリット
- **Kaliほどのコミュニティ規模はない**  
  Kali Linuxに比べると、ユーザー数・情報量ともにやや少ないです。
- **一部ツールの互換性やバージョンの遅れ**  
  ツールやリポジトリの更新状況によって、最新のペンテストツールを利用できない場合があります。
- **やや「何でも入り」な設計**  
  多機能である一方、必要ない機能も多く含まれてしまうことがあり、取捨選択が必要な場合があります。

---

## まとめ表

| 項目         | Kali Linux                          | Parrot OS                      |
|:------------ |:------------------------------------|:-------------------------------|
| ツール数     | 非常に多い（600+）                  | 多い（ただし種類がやや違う）    |
| プライバシー | 標準では重視していない               | 匿名性・プライバシーを重視      |
| 軽量性       | 標準的〜やや重い                     | 軽量（Liteあり）                |
| 用途         | ペンテスト専用                      | ペンテスト + 普段使いも対応      |
| 情報量       | コミュニティ・情報が非常に多い       | やや少なめ                      |

---

### 結論
- **Kali Linuxは純粋にペンテスト用途に特化したい場合や、最も広く使われている環境を求める人向け。**
- **Parrot OSはプライバシーや匿名性を重視したい場合や、ペンテスト以外も含めて多用途に使いたい場合におすすめです。**

# 攻撃用OSの仮想環境作成

MacOSを前提とし、UTMで仮想化する手順を示します。

## Parrot OS

### 1. Parrot OSのダウンロード

[Parrot OSのダウンロードページ](https://parrotsec.org)から最新版をダウンロードします。

1. `Choose Category` : Virtual
2. `Choose Edition` : Security
3. `Choose Architecture` : ARM64
4. `Download` : UTM (Apple Silicon)

### 2. UTMのインストール

[UTMのダウンロードページ](https://mac.getutm.app)からダウンロードします。

### 3. Parrot OSのインストール

UTMを起動し、＋ボタンからParrot OSのイメージファイルを選択します。
その後、右上の複製ボタンをクリックして、新しい仮想マシンを作成します。
※「この仮想マシン、およびその全てのデータを複製しますか？」 というダイアログが表示されたら、「はい」を選択してください。

この操作を実行後、イメージファイルを削除することができます。

### 4. Parrot OSの設定

仮想マシンを起動します。

**GRUB（ブートローダー）** のメニューで、Parrot OSの起動方法を選択する画面が表示されます。  
標準的には、次のように選ぶのが基本です。

| 項目名                                         | 用途・説明                                                                                         | 通常の選択       |
|:-----------------------------------------------|:--------------------------------------------------------------------------------------------------|:----------------|
| Parrot OS 6 GNU/Linux                         | 標準的な起動方法。グラフィカル環境が利用可能。                                                    | ◎（推奨）       |
| Text-only mode / Recovery                     | テキストモード（コマンドライン専用）や回復モード。トラブル時用。                                   | △（通常不要）   |
| Nouveau and Nvidia drivers OFF                | Nvidia関連のドライバを使わず起動。GPUトラブル時のセーフモード。                                    | △（通常不要）   |
| ACPI OFF and Nomodeset ON                     | ハードウェア互換性対策。起動できない場合のオプション。                                              | △（トラブル用） |
| Advanced options for Parrot OS 6 GNU/Linux    | 詳細なカーネル選択など。トラブルシューティング用。                                                  | △（通常不要）   |
| UEFI Firmware Settings                        | 仮想UEFI設定画面の起動。通常のOS起動には不要。                                                     | ×（不要）       |
